# ご当地グルメ手帖

会員のランク(閲覧のみ / 投稿可 / 編集可 / 管理人)に応じて、
できることが変わるご当地グルメまとめサイトです。

## 1. ローカルで動かして確認する(任意・パソコンにNode.jsが必要)

```bash
npm install
cp .env.local.example .env.local
# .env.local をSupabaseのURL・anonキーで書き換える
npm run dev
```

http://localhost:3000 で確認できます。

パソコンでの確認をスキップして、直接Vercelにデプロイしても問題ありません。

## 2. Vercelにデプロイする(無料)

1. このフォルダ一式をGitHubリポジトリにアップロードする
   (GitHubを使ったことがなければ「GitHub Desktop」というアプリが簡単です)
2. https://vercel.com にアクセスし、GitHubアカウントでログイン
3. 「Add New → Project」から、今アップロードしたリポジトリを選択
4. 「Environment Variables」の欄に以下の2つを追加
   - `NEXT_PUBLIC_SUPABASE_URL` … SupabaseのProject URL
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` … Supabaseのanon publicキー
5. 「Deploy」を押す。1〜2分で公開されたURLが発行されます

## 3. 最初の管理人になる

1. 公開されたサイトで、自分のメールアドレスで会員登録する
2. メールに届く確認リンクをクリックする
3. Supabaseの **SQL Editor** で以下を実行(メールアドレスを書き換える)

```sql
update profiles set role = 'admin'
where id = (select id from auth.users where email = 'あなたのメールアドレス');
```

4. サイトに再ログインすると、ヘッダーに「ランク管理」ボタンが表示されます

## 4. 他のユーザーのランクを変える

管理人でログインした状態で `/admin` ページを開くと、
会員登録した人の一覧とランク変更用のドロップダウンが表示されます。

## ランクの権限まとめ

| ランク | 閲覧 | 投稿 | 自分の投稿の編集/削除 | 他人の投稿の編集/削除 | ランク変更 |
|---|---|---|---|---|---|
| 閲覧のみ | ○ | - | - | - | - |
| 投稿可 | ○ | ○ | ○ | - | - |
| 編集可 | ○ | ○ | ○ | ○ | - |
| 管理人 | ○ | ○ | ○ | ○ | ○ |

権限は画面の表示だけでなく、Supabase側のRow Level Security(RLS)でも
二重にチェックされているので、URLを直接いじっても不正な操作はできません。

## 表示モード(一覧・地図・カレンダー)について

トップページ上部のタブで3つの表示を切り替えられます。

- **一覧**:都道府県・ジャンルで絞り込みできます
- **地図**:現在地を中心に表示し、投稿は写真付きのピンで表示されます
  - 初回アクセス時にブラウザが位置情報の利用許可を求めます。許可しない場合は東京駅を中心に表示されます
  - 地図上のピンは、投稿時に入力した「住所」から自動で計算しています(無料のOpenStreetMapという仕組みを使用)。住所が空欄だったり、位置を特定できなかった投稿は地図に表示されません
- **カレンダー**:今日が旬の投稿をまとめて表示し、月別カレンダーで「旬の日付」をピンポイント指定した投稿を確認できます

## 住所から位置情報が取得できない時

「お店の名前」だけで住所が曖昧だと、位置情報がうまく取得できないことがあります。
できるだけ詳しい住所(都道府県・市区町村・番地まで)を入力すると精度が上がります。
